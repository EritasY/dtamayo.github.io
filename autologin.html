Passwordless remote logins using public/private key pairs

We can automate the step of logging in with our username and password each time by creating a public / private key pair. You put your public key on as many workstations as you like, but you keep your private key on your own computer and never share it (would give others access to any workstations where you've put your public key). To create a key pair, we first open a terminal window. On Mac you would find this under Applications->Utilities. On Windows hit the windows key and search for Powershell (this will only work with post-2018 versions of Windows 10 or later). Then: 

                    
                        ssh-keygen -t rsa

On Windows this will first prompt you for where you want to put the file. Press enter to accept the default. Then on both Mac and Windows, it will prompt you for a passphrase. Press enter twice to NOT have a passphrase. This will create an id_rsa (private key) and id_rsa.pub (public key) file in the .ssh directory in your home folder (more on where that is below). We now need to paste the contents of id_rsa.pub into ~/.ssh/authorized_keys (a file containing the public keys of all authorized users) on each of the workstations you want to use. Copy pasting directly in terminal can cause problems with newlines--an easier way to do it by typing this command on your own computer (where you created your public/private key pair) to pass them through ssh. One difference is that on Mac and linux, the tilde ~ refers to your home directory, on Windows it's $HOME. On Mac we would use 

                    
                        cat ~/.ssh/id_rsa.pub | ssh username@sphinx.physics.hmc.edu 'cat >> ~/.ssh/authorized_keys'

where again you should replace 'username' with your own username. On Windows you would use: 

                    
                        cat $HOME/.ssh/id_rsa.pub | ssh username@sphinx.physics.hmc.edu 'cat >> ~/.ssh/authorized_keys'

This will prompt you for your password like before, but once done, if you now exit out and try to ssh again, you should get logged in automatically! If you also want to automate logins into our other workstations (and you do!), that's a bit more complicated, and you should do it by setting up an ssh config file: Simplifying advanced ssh commands with an ssh config file.
