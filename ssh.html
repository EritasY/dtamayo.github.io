Logging into a workstation remotely

We can log into our workstations remotely using ssh (secure shell). First we need to open a terminal window. On Mac you would find this under Applications->Utilities. On Windows hit the windows key and search for Powershell (this will only work with post-2018 versions of Windows 10 or later). You might add a shortcut since we'll use the terminal a lot. Now if you wanted to log into our workstation sphinx.physics.hmc.edu, you would type into the terminal and press enter: 

                    
                        ssh username@sphinx.physics.hmc.edu

where you should replace 'username' with the one you use on our workstations (i.e., your HMC username). This will prompt you for the same password you would use when logging into the workstation in person. Once logged in, it's just as if your terminal were a terminal that you opened at the workstation itself--you can run any linux commands you want.

For security reasons, sphinx is the only workstation connected to the outside world. It is an old computer only meant to process logins and pass network traffic along. To log in to one of the other workstations (e.g., hr8799.physics.hmc.edu) that we would actually want to use for calculations, we need to log into sphinx first. So we would log into sphinx as above, and once our terminal is running commands on sphinx, we would do, e.g., 

                    
                        ssh hr8799.physics.hmc.edu

Here we can omit our username, since our username is the same on sphinx and hr8799. Note that if you tried the command above from your own laptop, it wouldn't work. You need to log into sphinx (our login node) first.

Now you can do anything you would do in a terminal remotely, e.g., you could use any of the linux text editors to write a script.py file and run it. If you want to use a jupyter notebook, check out Running a Jupyter notebook remotely.

Since we're going to have to do these logins over and over, you should definitely make your life more convenient by following the Passwordless remote logins using public/private key pairs and Simplifying advanced ssh commands with an ssh config file tutorials.
